# Role-Specific CV Audit

Use this reference when the user wants to tailor a CV for different role directions.

## Goal

Do not audit only which project bullets to keep. Also audit:

- which source entries should appear
- which should be shortened, weakened, or dropped
- which items belong in the Skills section
- which items should stay inside project bullets instead of entering Skills

## Typical Role Directions

### Agent / LLM Engineering

Prioritize:

- agent runtime work
- tool calling
- RAG
- evaluation and safety boundaries
- productized infra and systems integration

Skills section should emphasize:

- agent systems
- RAG
- evaluation
- Python / TypeScript / APIs / testing / systems

### Data Science / Applied AI

Prioritize:

- measurable model or experiment outcomes
- data pipelines
- retrieval, ranking, classification, forecasting, or model evaluation
- decision-relevant metrics

Skills section should emphasize:

- modeling
- experimentation
- metrics
- Python / SQL / analysis tooling

### CV / Multimodal

Prioritize:

- computer vision datasets
- benchmark results
- multimodal pipelines
- visual or embodied modeling

Skills section should emphasize:

- computer vision
- multimodal modeling
- dataset construction
- evaluation metrics

## Audit Output Contract

When producing a role-specific audit, include:

1. target role direction
2. must-keep entries
3. optional entries
4. weaken/drop entries
5. Skills section keep list
6. Skills section weaken/drop list
7. notes for 1-page CV
8. notes for 2-page CV

## Heuristic

- a strong project bullet does not automatically belong in the Skills section
- a strong skill should usually appear in more than one entry or be critical for the target role
- if a skill is only a one-off implementation detail, keep it inside the project bullet instead of the Skills block
