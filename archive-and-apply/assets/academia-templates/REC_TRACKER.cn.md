# Recommendation Letter Tracker / 推荐信追踪表

> 用途：管理每位推荐人的关系维护、请求时机、提交状态。
> 状态：模板

## 使用原则

- 推荐信是申请中最不可控的环节，提前规划，提前联系。
- 每位推荐人维护一张表：关系深度、学术亮点、什么时候联系、提交状态。
- 建议提前 1-2 个月开始联系， deadline 前 2 周做最后跟进。
- 提交后记录提交时间和收到的确认。

## 推荐人评估维度

在请求之前，先评估每位推荐人对你的了解程度：

- **了解深度**：这节课的成绩 / 项目参与程度 / 合作时间
- **能写的亮点**：你的哪些具体工作 / 品质他们能举出例子
- **写作质量**：他们是否愿意写个性化推荐信，还是只会提交表单
- **提交可靠性**：这学期是否在校 / 是否及时 / 是否靠谱

## 推荐人来源

| 推荐人类型 | 适合项目 | 权重 |
|---|---|---|
| 科研导师 / 实验室老板 | PhD / 研究型硕士 | 核心推荐 |
| 课程教授 | 所有人 | 补充技术基础 |
| 实习主管 | MBA / 职业型硕士 | 补充职业能力 |
| 项目负责人 | 所有人 | 补充团队合作 |
| 社会活动负责人 | MPA / 公共服务类 | 补充服务动机 |

## 推荐结构

```md
# Recommendation Letter Tracker

## 推荐人 A

### 基本信息
- 姓名：
- 身份：教授 / 副教授 / 讲师 / 实习主管 / 项目负责人
- 机构：
- 联系方式：
- 认识多久：通过什么认识的

### 能写的亮点
- （从 source entries 提取，他能举出具体例子的）
- ...

### 关系维护记录
- YYYY-MM-DD: [事件]
- ...

### 请求与提交状态
- 打算请求：YYYY-MM-DD
- 实际请求：YYYY-MM-DD
- Deadline：YYYY-MM-DD
- 提交院校数：
- 提交状态：
  - [ ] School 1 — 提交 / 未提交
  - [ ] School 2 — 提交 / 未提交

### 跟进记录
- YYYY-MM-DD: 提醒邮件发送
- YYYY-MM-DD: 收到提交确认
```

## 注意事项

- 请求时提供 CV + SOP 草稿，方便他们写针对性的内容
- 至少提前 2 周请求，给自己留缓冲
- 如果推荐人说"不太了解你的研究"，换人
- 不要让推荐人提交他们无法说出具体例子的信

## 同步清单

- [ ] 推荐人已评估并确认能写个性化推荐信
- [ ] 提前 1-2 个月开始联系
- [ ] 提供 CV + SOP 草稿给推荐人
- [ ] Deadline 前 2 周做最后跟进
- [ ] 每封提交后记录状态
