# 条目模板

这个模板用于维护 `experiences/`、`projects/`、`publications/` 下的源条目。原则是先把事实写全，再从这里提炼简历、面试稿和定制材料。

## 使用原则

- 默认用中文写，英文信息只在必要处保留原名
- 先写“源事实”，不要一开始就压缩成简历 bullet
- 能量化就量化，不能确定就先空着，不要编
- 项目、经历、论文共用一套大结构，但允许按类型增删字段

## 推荐文件头

标题直接使用条目名称，例如：

```md
# 条目标题示例
```

```md
# 条目名称

- 时间：
- 角色：
- 团队 / 机构：
- 导师 / 合作者：
- 方向：
- 相关链接：
  - 

## 一句话概述

用 1 到 2 句话说明：这是一个什么场景，我解决了什么问题，我主要做了什么。

## 背景与目标

说明这个条目的背景、目标用户或研究问题，以及为什么值得做。

## 我的工作

- 我实际负责了什么
- 我的边界在哪里
- 我交付了哪些具体模块、工具、流程或实验

## 技术方案

- 用了哪些模型、框架、系统、工具链
- 数据如何组织、清洗、训练、评测、部署
- 关键工程设计、架构权衡、接口约束、风险控制

## 结果与产出

- 指标、排名、论文、上线效果、测试覆盖、用户价值、可交付物

## 难点与复盘

- 最难的问题是什么
- 你是怎么处理的
- 这件事带来了什么可复用经验

## 技术关键词

`Keyword1` `Keyword2` `Keyword3`

## 后续可补充

- [ ] 数据规模 / 用户规模 / 文档规模
- [ ] 定量指标
- [ ] 我的具体边界
- [ ] 架构图 / 流程图 / 系统图
- [ ] 面试里可展开的问题
```
